# 缩短宽屏保护设计

## 背景

脚本应用默认宽屏后，会启动短期保护，防止播放器初始化把 `data-screen` 重置为 `normal`。当前保护持续 12 秒，并最多重新应用宽屏 3 次，导致用户在进入页面后的数秒内手动退出宽屏时，脚本又将宽屏打开。

## 方案

- 将保护窗口从 12 秒缩短为 3 秒。
- 将保护期内重新应用目标模式的上限从 3 次降低为 1 次。
- 保持 800 毫秒轮询间隔及现有 `MutationObserver` 机制不变。
- 不增加用户点击或 `Escape` 键识别逻辑，不改变其他播放器模式行为。
- 更新 README 中默认播放器模式的说明，使保护行为可被用户理解。

## 预期行为

播放器初始化偶尔将目标模式重置为常规模式时，脚本仍可恢复一次。恢复一次后，用户再次退出不会被重新拉回；无论是否发生恢复，保护最多持续约 3 秒。

## 验证

- JavaScript 语法检查通过。
- 静态检查确认保护窗口为 3000 毫秒、重新应用上限为 1 次。
- Git 差异只包含上述参数、相关注释和 README 说明。
